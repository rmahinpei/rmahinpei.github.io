Most of my recent learnings and interests have focused on formal methods and interactive theorem provers, especially the practical applications of formal methods and the potential use of interactive theorem provers for educational purposes.

At a high-level, formal methods apply mathematical thinking to computational systems. This usually involves rigorously specifying a system using a modelling language and then formally verifying the specified properties of that system. A common application of formal methods is to verify that hardware and software systems are error-free. Although slightly outdated, Michael Collins's article on formal methods is a great introduction on what formal methods are all about and is a highly recommended read for anyone who's new to this topic.

Below is a recommended list of talks and papers relating to the applications of interactive theorem provers, especially for educational purposes.

Recommended Talks

  • Propositions as Types: A 2015 talk by Philip Wadler on the correspondence between mathematical logic and programming languages. The main takeaway was that propositions, proofs, and similification of proofs in logic accordingly correspond to types, programs, and evaluation of programs in programming languages.
  • Teaching Proofs to Computers: A 2020 talk by Kevin Buzzard on the use of interactive theorem provers within mathematics. Buzzard is a huge advocate for using proof assistants, such as Lean, as a part of undergraduate mathematics curriculums in order to enable students to better understand proof structures and get automatic feedback on their work.
  • Formal Methods for the Informal Engineer - The Z3 Theorem Prover: A 2021 talk by Phil Zucker on the Z3 Theorem Prover. Although the talk mainly focuses on using Z3 via a Python API, there are also small sections discussing what formal methods are all about and how theorem provers work.
  • Formal Methods for the Informal Engineer - The Coq Theorem Prover: A 2021 talk by Cody Roux on the Coq Theorem Prover. The contents in this talk are much more technical compared to the talk above since the main focus is on learning the major features of Coq and then using Coq to prove some example propositions.
